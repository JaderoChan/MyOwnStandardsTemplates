# Markdown 文档空格标准

Markdown 文档编写中需要通过一个空格对以下内容进行分隔。（以下内容中出现的 **正文** 均指代中文正文（因为在英文正文中，空格本身就常用于各种内容之间的分隔），且不包含标点符号（无论半角还是全角符号））

## 必须

- 西语字母、单词与正文

    **例外情况**

    - 当**单个字符**作为某个对象的指代符号或实例符号时

        - `实例A为实例B的基类` 适用此种情况。（A、B 均为单个英文字符；A、B 均为 **实例** 这一概念的实例）
        - `类Animal为类Dog的基类` 不适用此种情况，故为错误写法。正确写法为 `类 Animal 为类 Dog 的基类`。（Animal 与 Dog 均非单字符。此外即使后者非 Dog，而是 A（假设为某个动物的名称），出于与前者保持一致性的原因，应该仍然使用空格对其进行分隔。）

    - 当某个西语字母或单词与某个对象是强相关的，互相绑定时

        - `此文档的作者是𬱖珞JaderoChan` 适用此种情况。（𬱖珞与 JaderoChan 是强相关，互相绑定的）

- 代码片段与正文
- 公式片段与正文

## 建议

- 文档高亮或强调部分与正文
- 使用引号包围的部分与正文
